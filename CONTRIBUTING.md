# Contributing

Contributions are welcome. Please follow these guidelines.

## Reporting Issues

- Search existing issues before opening a new one
- Include your OS, Node.js version, and package manager
- Paste the exact output from the checker/fixer

## Submitting Changes

1. Fork the repository
2. Create a branch: `git checkout -b fix/your-fix` or `feat/your-feature`
3. Make your changes
4. Test with `create-test-env.sh` to verify checker behavior
5. Open a pull request with a clear description of what changed and why

## Scope

This tool targets a specific supply chain attack. Please keep contributions focused:

- Bug fixes and detection improvements are welcome
- New package manager support is welcome
- Unrelated features or scope creep will not be merged

## Security Issues

Do not open a public issue for security vulnerabilities. See [SECURITY.md](SECURITY.md).
